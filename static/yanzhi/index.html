EDITIO ALTERA · MMXXVI
A Hangul-Based Script for Min, Wu & Other Sinitic Tongues
臺語 · 蘇州話 · 私人音位字母表

此版重整為「通用框架 + 兩語子集」三層結構。原版以臺語為主、蘇州話留位；今將兩者並列為對等子集，皆建於同一套通用音位骨架之上。後若添上海、溫州、福州諸腔，亦不過再添子集，無需改動底層。

Chapter I

通用原則

  1. 零自造字形。凡聲母元音韻尾，盡用現行（含古）韓文字母；至多以下加附號（如下加點）為之區別。
  2. 緊音即濁音。韓文緊音位 ㄲ ㄸ ㅃ ㅆ ㅉ 重新定義為漢語之濁聲母 g d b z dz。雙字母＝濁化，此為通用規則。
  3. ㅇ 即 ŋ。聲母位、韻尾位一律讀為軟顎鼻音。零聲母不著字，元音獨成方塊。
  4. 下加點即擴充。某子集需新增之聲母（如蘇州 f v ɦ，巴盟卷舌組），於既有字母下加一點以為之，避免造字。
  5. 聲調以數字。上標阿拉伯數字 1–8 記八聲本調，變調不記。
  6. 入聲韻尾同形於聲母。ㅂ ㄷ ㄱ 居方塊之下即為 -p -t -k 入聲，ㅎ 則為喉塞 -ʔ。
  7. 子集分立。各語各持其聲母、元音、韻尾、聲調之表。同一字母在不同子集可有不同音值（如 ㅐ 於臺為 ai、於蘇為 ɛ），讀寫時須先聲明語別。

七則之中，二、三、四、七為本版骨幹。第二、三則承自初版；第四則為本版新增之擴充機制；第七則為本版結構上之核心宣告——同字異讀，因語而異。

何以採子集而非疊加？蓋臺、蘇二語雖同屬南方漢語，元音系統與韻尾系統之差異甚大：臺語有 -p -t -k -m 諸古韻尾、有鼻化元音；蘇州則無此諸般，反多 y ø ɛ 三圓不圓之精細區分。若強合為一表，必處處須記「此於臺如何、於蘇如何」；不如分立兩表，各得其全。

Chapter II

通用框架 · 音節骨架

任何漢語音節，於本系統中皆解為下列五位之填空：

聲調 · [ 聲母 + 介音 + 主元音 + 韻尾 ]方塊 + ~可選鼻化

字母倉庫

本系統所動用之韓文字母，可分為三類：

  • 現代基礎字母（19+21）：聲母 ㅂㅍㅁㄷㅌㄴㄹㅈㅊㅅㄱㅋㅇㅎ 等 19 個，元音 ㅏㅑㅓㅕㅗㅛㅜㅠㅡㅣ 等 21 個。皆主力。
  • 緊音重疊字母（5）：ㄲ ㄸ ㅃ ㅆ ㅉ。原為韓語緊音，本系統重新定義為漢語濁聲母 g d b z dz。
  • 下加點擴充字母：於基礎字母下方加一小點為記，用以表本系統未及之輔音。如 ㅍ = f、ㅂ = v、ㅎ = ɦ、ㅈ ㅊ ㅅ = 卷舌組（巴盟用）。此實現以 CSS 渲染此點，避開字體不支援組合附號之疑；於純文字場合可改以 U+0323 組合下點或他法為之。此擴充機制可無限延伸而不傷骨幹。

Chapter V

組字 · 方塊之法（兩語共用）

方塊組字沿韓文之法：聲母在左，元音在右或在下，韻尾置於底，聲調寫於方塊右上外。下加點屬於聲母之一部分，與聲母共佔左位。

臺語絕大多數音節一方塊可成；蘇州亦然，且因無 iong 之類複合，反而更乾淨。然臺語 iong 韻（如「中 tiong」之韻）因 Hangul Syllables 區無此預編碼，須拆為兩方塊 됴ㅇ——此為系統已知讓步。

❋ ❋ ❋

向後可擴 · 未盡之語

本框架之下尚可繼續加入：

  • 上海話：與蘇州子集九成共用，差別主要在聲調系統（上海七調已簡化為兩級陰陽），可視為蘇州子集之變體。
  • 溫州話：保留更多濁聲母、更多元音、複雜變調，需擴充新元音字（或可借 ㅑ ㅕ ㅛ ㅠ 之餘位）。
  • 福州話：聲母類化、元音變讀為其特色，框架可承之，但須註明「連讀類化規則」於子集首。
  • 巴盟話 / 西北官話：需卷舌組（ㅈ̣ ㅊ̣ ㅅ̣），下加點機制已備。
  • 粵語：可全用通用韻尾倉庫七位（-p -t -k -m -n -ŋ -ʔ 皆有），九調可全部啟用 1–9（需加調 9 位）。

凡新增之語，重在三事：定其聲母表、定其元音重音值、定其韻尾子集。骨架不動。
